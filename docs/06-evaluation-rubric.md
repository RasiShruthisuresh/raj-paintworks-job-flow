# Evaluation Rubric

## Product Judgment

- Understands Raj's small-business context.
- Keeps workflows simple and operational.
- Makes sensible tradeoffs when scope is large.

## Agentic Engineering

- Uses coding agents effectively without outsourcing judgment.
- Breaks work into reviewable steps.
- Catches and corrects agent mistakes.
- Documents agent usage honestly.

## Code Quality

- Fixes root causes rather than masking symptoms.
- Uses a coherent data model.
- Handles validation and errors consistently.
- Avoids broad rewrites that introduce unnecessary complexity.
- Adds useful tests or a strong manual QA checklist.

## Feature Completeness

- Lead capture works.
- Stage movement works.
- Timesheet tracking works and persists.
- Analytics dashboard is data-driven.
- Railway deployment works.

## UI Quality

- Follows the Airbnb-inspired design guide.
- Works on desktop and mobile.
- Uses clear hierarchy and readable data displays.
- Provides useful empty, loading, and error states.

## Deployment

- Private GitHub repository is accessible to the interviewer.
- Railway URL is live.
- Environment variables and setup steps are documented.
- Docker or direct Railway build is reproducible.
